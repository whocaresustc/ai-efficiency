## 1. 交互模式

### 1.1 三种模式

| 模式 | 说明 | 行为 |
|------|------|------|
| 默认模式 | 最谨慎 | 文件创建、修改、删除及终端执行都需逐次确认 |
| 自动模式 | 提示语：`accept edit on` | 文件写入和编辑自动通过；终端命令仍可能因安全原因弹出确认（如 `mkdir`、`npm install`） |
| 规划模式 | 提示语：`plan mode on` | 只讨论方案，不执行文件修改，也不执行终端命令；适合先梳理架构、任务拆解、技术路线 |

### 1.2 模式切换

- `Shift + Tab`：在默认模式、自动模式、规划模式之间循环切换

### 1.3 危险权限模式

- 启动参数：`--dangerously-skip-permissions`
- 作用：跳过文件写入和终端命令的确认
- 风险：模型拥有与当前用户同级别的终端权限，误操作可能造成较大破坏
- 适用前提：仅在可信环境、明确追求效率时使用

## 2. 输入与编辑

| 操作 | 方式 |
|------|------|
| 换行 | `Shift + Enter` |
| 提交请求 | `Enter` |
| 用 VS Code 编辑长 prompt | `Ctrl + G`（依赖本机已安装 VS Code，保存并关闭标签页后内容同步回终端） |
| 直接执行系统命令 | 输入框开头加 `!`，例：`!open index.html` |
| 粘贴图片 | `Ctrl + V`（macOS 上也用 `Ctrl + V`，不是 `Command + V`） |
| 退出程序 | 连按两次 `Ctrl + C` |

## 3. 任务管理与后台执行

### 3.1 为什么需要后台任务

- `npm run dev` 这类持续运行命令会阻塞当前交互
- 若不转后台，终端输入会被占住，无法继续与 Claude Code 对话

### 3.2 操作方式

| 操作 | 方式 |
|------|------|
| 将阻塞式任务转入后台 | `Ctrl + B` |
| 查看后台任务列表 | `/tasks` |
| 结束任务 | 在任务列表中按 `K` |
| 返回主界面 | `Esc` |

## 4. 上下文控制与会话管理

| 功能 | 命令/快捷键 | 说明 |
|------|------------|------|
| 压缩上下文 | `/compact` | 压缩历史上下文，降低 Token 消耗，提升后续响应效率；长对话、高频迭代后尤其重要 |
| 查看压缩结果 | `Ctrl + O` | 查看压缩后的上下文内容 |
| 清空上下文 | `/clear` | 彻底清除会话历史 |
| 恢复历史会话 | `/resume` | 列出历史会话并手动选择恢复 |
| 自动继续上次会话 | 启动参数 `-c` | `-c` 是 `continue` 的缩写 |

## 5. 视觉能力与 MCP 联动

### 5.1 图片识别

- 可直接把设计稿图片拖拽到终端
- 也可复制图片后按 `Ctrl + V` 粘贴
- macOS 上粘贴图片也要用 `Ctrl + V`，不是 `Command + V`

### 5.2 MCP（Model Context Protocol）

- MCP 是模型连接外部工具的标准通道
- 用于让 Claude Code 与 Figma 等外部系统交换上下文和工具结果
- 使用 `/mcp` 查看和管理已安装的 MCP 工具

## 6. CLAUDE.md 与 Memory 持久化

### 6.1 CLAUDE.md 的作用

- 保存项目背景、开发规范、长期约束和注意事项
- 每次启动 Claude Code 时会自动读取

### 6.2 创建与管理

| 命令 | 作用 |
|------|------|
| `/init` | 初始化生成相关文件 |
| `/memory` | 打开 memory 管理入口 |

- 可区分项目级和用户级 CLAUDE.md

### 6.3 使用场景

- 固化团队规范
- 固化项目边界
- 固化代码风格、目录约定、工作方式

## 7. Hooks 自动化

### 7.1 定位

- Hook 是工具执行前后触发的自动化逻辑
- 适合做格式化、通知、后处理等动作

### 7.2 配置流程

1. 输入 `/hook`
2. 选择触发时机，如 `post-tool-use`
3. 选择目标工具，如 `write`、`edit`
4. 编写执行逻辑

### 7.3 配置层级

| 配置文件 | 范围 | 说明 |
|----------|------|------|
| `settings.local.json` | 本机项目级 | 自动加入 `.gitignore`，避免误提交 |
| `settings.json` | 项目级 | 可随 Git 共享 |
| 用户级配置 | 全局 | 存放于用户目录，对所有项目生效 |

### 7.4 实战示例

- 通过 Hook 接入 Prettier，实现代码改写后的自动格式化
- 依赖系统中已有 `jq` 和 Prettier 等工具
- 若本机缺少这些依赖，Hook 将无法正常工作

## 8. Skills、SubAgent、Plugin

### 8.1 Skill

- 本质上是动态加载的任务说明书或 Prompt 规范
- 适合处理特定类型任务，例如日报、设计规范、定制工作流
- 自定义路径：`~/cloud/Skills/<Skill名称>/Skill.md`
- 触发方式：Claude Code 可自动识别并触发，也可由用户主动调用

### 8.2 SubAgent

- 具有独立上下文和独立工具权限的子智能体
- 最大价值：把中间推理过程与主会话隔离，只把结果返回主对话

**适用场景：**
- 万行代码审查
- 大范围重构分析
- 重型任务拆分

**创建方式：**
1. 输入 `/agents`
2. 选择创建项目级或用户级 agent
3. 配置模型与工具权限
4. 编写对应的 `.md` 规范

### 8.3 Plugin

- 一键安装包，视频把它描述成"全家桶"
- 可封装并分发 Skill、SubAgent、Hook、MCP 的组合能力
- 输入 `/plugin` 进入插件管理器，常见入口包括 `Discover`、`Installed`、`Marketplace`

### 8.4 五者关系总览

| 组件 | 职责 |
|------|------|
| MCP | 连接外部工具 |
| Hook | 在流程节点上自动执行动作 |
| Skill | 补充任务规范和提示策略 |
| SubAgent | 把复杂任务拆到独立上下文中处理 |
| Plugin | 把以上能力打包、安装和复用 |

## 9. 回滚与版本管理

### 9.1 回滚方式

- `/rewind`
- 或连按两次 `Esc`

### 9.2 回滚边界

- **可以回滚：** Claude Code 直接写入或修改的内容
- **无法回滚：** 通过终端命令产生的副作用，例如：
  - `mkdir` 创建的目录
  - `npm install` 下载的依赖
  - 其他由 shell 命令间接生成的文件或状态

### 9.3 最佳实践

- 必须配合 Git 使用
- Claude Code 的回滚适合局部撤销，不适合作为完整版本控制替代

## 10. 实战案例：Todo 应用完整链路

| 阶段 | 做法 |
|------|------|
| **原型** | 让 Claude Code 生成单文件 HTML 版 Todo 应用，快速得到可运行原型 |
| **规划重构** | 切换到 Plan Mode，讨论改造为 React + TypeScript + Vite 架构 |
| **需求细化** | 用 `Ctrl + G` 打开外部编辑器，补充 UI 风格一致性、优先级区分、颜色标识等需求 |
| **环境准备** | Auto Mode 下自动创建目录和修改文件，高风险命令（如 `npm install`）人工确认 |
| **服务启动** | 执行 `npm run dev`，按 `Ctrl + B` 放入后台，保持服务运行的同时继续调整代码 |
| **UI 还原** | 借助 Figma MCP 获取设计稿上下文和截图，推动更精确的界面实现 |
| **工程化** | 使用 Hook 自动执行 Prettier，减少手动格式化成本 |
| **风格优化** | 安装 front-end design 插件，调用设计类 Skill 提升配色和排版质量 |

---

## 11. 最佳实践总结

1. 复杂任务先用 Plan Mode 梳理方案，再开始落地
2. 高频修改和长链路对话后，及时执行 `/compact`
3. 遇到阻塞式命令，立刻用 `Ctrl + B` 放后台
4. 对高风险自动化保持边界意识，谨慎使用 `--dangerously-skip-permissions`
5. 用 CLAUDE.md 固化长期上下文，减少重复解释
6. 对大任务优先使用 SubAgent，避免主上下文被中间过程污染
7. 用 Hook 承接格式化等机械步骤，把 Claude Code 用在更高价值的位置
8. 全量回滚与审计仍应依赖 Git

---

## 12. 易忽略细节

- 输入框下方的 `? for shortcuts` 只是交互提示，不是独立模式
- `accept edit on` 不等于所有操作都自动放行，终端命令仍可能要求确认
- `Ctrl + G` 的前提是本机装有 VS Code
- macOS 粘贴图片时也要用 `Ctrl + V`
- 后台任务结束时报错，不一定表示实际关闭失败
- 视频转录中对命令名有 `claude` / `cloud` 两种写法，整理时保留其原始表述
